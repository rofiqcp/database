# Features

Complete list of features available in the Elasticsearch Learning Module.

## Backend Features

### ✅ Database Operations
- **Elasticsearch Integration**: Distributed search and analytics engine with @elastic/elasticsearch client
- **Full-Text Search**: Powerful search with fuzzy matching and relevance scoring
- **Auto-initialization**: Index and mappings created automatically on first run
- **Schema Management**: Explicit mappings with proper field types
- **Data Persistence**: All data stored in Elasticsearch cluster
- **Near Real-Time**: Documents searchable within ~1 second

### ✅ API Endpoints
- **GET /api/data**: List all items
- **GET /api/data/:id**: Get single item by document ID
- **POST /api/data**: Create new item
- **PUT /api/data/:id**: Update existing item
- **DELETE /api/data/:id**: Delete item
- **POST /api/search**: Full-text search with fuzzy matching and filters
- **GET /health**: Health check endpoint

### ✅ Search Capabilities (Elasticsearch Strengths)
- **Fuzzy Search**: Handles typos and approximate matching (e.g., "laptpo" finds "laptop")
- **Multi-Field Search**: Search across name and description simultaneously
- **Relevance Scoring**: Results ranked by relevance using TF-IDF / BM25
- **Category Filtering**: Exact-match filtering on keyword fields
- **Price Range Filtering**: Range queries with gte/lte operators
- **Combined Queries**: Bool queries combining must, filter, should clauses

### ✅ Security & Validation
- **Input Validation**: Using validator library
- **Error Handling**: Comprehensive error handling
- **CORS Support**: Configurable CORS for frontend
- **Data Sanitization**: Automatic trimming and cleaning

### ✅ Developer Experience
- **Hot Reload**: Nodemon for development
- **Environment Variables**: Dotenv configuration
- **Logging**: Request logging middleware
- **Consistent Response Format**: Standardized JSON responses
- **Error Messages**: Clear, actionable error messages

## Data Features

### ✅ Data Model
```javascript
{
  id: String (auto-generated by Elasticsearch),
  name: String (text + keyword, required),
  description: String (text, optional),
  category: String (keyword, optional),
  price: Number (float, default: 0),
  quantity: Integer (default: 0),
  created_at: Date (ISO string, auto),
  updated_at: Date (ISO string, auto)
}
```

### ✅ Data Operations
- **Pagination Ready**: Can be implemented with from/size
- **Sorting**: Ordered by creation date
- **Filtering**: Multiple filter criteria
- **Full-text Search**: Powerful search across fields with fuzzy matching
- **Aggregations Ready**: Can add analytics and statistics
