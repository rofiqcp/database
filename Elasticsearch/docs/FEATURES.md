# Features

Complete list of features available in the Elasticsearch Learning Module.

## Backend Features

### ✅ Database Operations
- **Elasticsearch Integration**: Distributed search engine with @elastic/elasticsearch driver
- **Auto-initialization**: Index created automatically with proper mappings on first run
- **Schema Management**: Explicit field mappings with text, keyword, float, integer, date types
- **Full-Text Search**: Powered by Elasticsearch inverted index and analyzers

### ✅ API Endpoints
- **GET /api/data**: List all items (match_all query)
- **GET /api/data/:id**: Get single item by document ID
- **POST /api/data**: Create new item (client.index)
- **PUT /api/data/:id**: Update existing item (client.update)
- **DELETE /api/data/:id**: Delete item (client.delete)
- **POST /api/search**: Search and filter with query DSL (multi_match, term, range)
- **GET /health**: Health check endpoint

### ✅ Security & Validation
- **Input Validation**: Using validator library
- **Query Injection Prevention**: @elastic/elasticsearch client escapes parameters
- **Error Handling**: Comprehensive error handling
- **CORS Support**: Configurable CORS for frontend
- **Data Sanitization**: Automatic trimming and cleaning

### ✅ Developer Experience
- **Hot Reload**: Nodemon for development
- **Environment Variables**: Dotenv configuration
- **Logging**: Request logging middleware
- **Consistent Response Format**: Standardized JSON responses
- **Error Messages**: Clear, actionable error messages

### ✅ Data Display

## Data Features

### ✅ Data Model
```javascript
{
  id: String (auto-generated by Elasticsearch),
  name: String (required, text + keyword),
  description: String (optional, text),
  category: String (optional, keyword),
  price: Number (default: 0, float),
  quantity: Integer (default: 0),
  created_at: Date (auto, ISO 8601),
  updated_at: Date (auto, ISO 8601)
}
```

### ✅ Data Operations
- **Pagination Ready**: Can be implemented with from/size
- **Sorting**: Ordered by creation date descending
- **Filtering**: Multiple filter criteria via bool query
- **Full-text Search**: Multi-match with fuzziness across fields
- **Relevance Scoring**: Elasticsearch ranks results by relevance
